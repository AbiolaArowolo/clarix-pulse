# Clarix Pulse - Quick Onboarding

**Document Date**: `2026-03-29 -04:00`

## Brief Step-By-Step Guide

### For a new customer

1. Open [pulse.clarixtech.com](https://pulse.clarixtech.com/).
2. Register the company with email and password.
3. Clarix Pulse generates a 365-day access key.
4. If SMTP is configured, the key is emailed automatically.
5. If SMTP is unavailable, the key is shown once in the registration flow as a fallback.
6. New accounts start disabled by default.
7. A platform admin enables the account from the admin screen.
8. Sign in with email, password, and the access key.
9. Open the dashboard's `Onboarding` page.

### For a new Windows node

1. While signed in, download the installer bundle from the onboarding or account page, or generate a secure expiring installer link if hub signing is configured.
2. Copy the bundle to the Windows PC and unzip it.
3. Start the monitored application if possible.
4. Run:

```powershell
powershell -ExecutionPolicy Bypass -File .\discover-node.ps1
```

5. In the dashboard, open `Remote Setup` and upload the discovery report.
6. Review the node and player details.
7. Click `Provision node and download config`.
8. On the node, start the local setup experience with `configure.bat`.
9. Import the downloaded `config.yaml`, or paste the secure config link generated by `Remote Setup` when hub signing is configured.
10. Click `Save Local Settings`.
11. Run `install.bat`.

### After install

1. Confirm the service is running:

```bat
sc query ClarixPulseAgent
```

2. Confirm the node appears in the signed-in workspace.
3. Confirm alert contacts belong to that workspace.

## Notes

- new dashboards start empty by default
- the registration email is the default alert recipient until changed
- installer and config pulls can be initiated only by signed-in enabled users
- secure installer/config links are expiring URLs created from the dashboard for node-side pulls
- the provisioned `config.yaml` flow is preferred over enrollment-key setup
- the enrollment key still exists as a fallback on the account and onboarding screens
