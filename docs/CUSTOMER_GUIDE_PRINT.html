Clarix Pulse

Customer Guide

This guide is written for customer operators. It explains how to get your first node online, how to finish setup on the Windows machine, and how to use the dashboard after the node starts reporting.

What Clarix Pulse does

Clarix Pulse uses a Windows node agent plus a central dashboard. You prepare the node from the dashboard, finish the local setup on the Windows machine, and then monitor the node from the live hub.

1. What You Need Before Starting

  • Your Clarix Pulse email, password, and access key.
  • A Windows machine where the node software will be installed.
  • Access to the monitored playout application if possible while it is running.
  • An enabled customer account. New accounts may need a platform admin to enable them first.

2. Sign In And Open Onboarding

Login Page

Enter your email, password, and access key, then click Sign in.

Use Forgot password if you cannot remember your password.

Onboarding Page

This is the easiest place to start your first node. It shows the installer, the setup flow, and the fallback enrollment key.

3. Get The Installer To The Windows Node

Download installer

Downloads the Windows bundle directly in your signed-in browser.

Create secure link

Creates an expiring direct download link for the installer.

Copy secure link

Copies the installer link so another operator can use it on the Windows node.

After the bundle is downloaded, move it to the Windows machine and unzip it.

4. Run Discovery On The Windows Node

Discovery helps Clarix Pulse detect likely players, logs, and paths before you provision the node.

Run this command on the Windows node

powershell -ExecutionPolicy Bypass -File .\discover-node.ps1

This creates the discovery report that you will upload in the dashboard.

5. Provision The Node In Remote Setup

  1. Open the dashboard.
  2. Find the Remote Setup panel.
  3. Click Upload discovery report.
  4. Review the node identity fields and each player block.
  5. Click Provision node and download config.

What Remote Setup does

It creates the node in your workspace, rotates a fresh agent token, mirrors the configuration to the hub, and downloads the final config.yaml.

Optional handoff

After provisioning, you can click Create install handoff page and send that temporary page to a field operator who will finish the Windows-side setup.

6. Finish The Setup On The Windows Node

  1. Run configure.bat.
  2. Use Upload report or config to import the downloaded config.yaml.
  3. Or paste the secure config link into Pull Setup From URL and press Pull from link.
  4. Review the imported values.
  5. Click Save Local Settings.
  6. Run install.bat.
Important

Saving local settings is required. Provisioning in the hub does not finish the install by itself. The Windows node must still save the final config locally and install the service.

7. Confirm The Node Is Live

  • Return to the dashboard.
  • Confirm the node appears under the correct site.
  • Check that the last heartbeat is updating.
  • Confirm player badges, connectivity, and stream thumbnails look correct.

8. Daily Monitoring Basics

Alarm banner

When a live alarm is active, the red banner appears at the top. Use Enable sound if your browser blocked audio, or Turn off alarm sound to silence the tone.

Per-player controls

Use Maintenance on/off to pause alerts during engineering work. Use Monitoring on/off only when you intentionally want that player out of live alarming.

9. Common Mistakes To Avoid

  • Do not stop at provisioning. The Windows node must still save the config and install the service.
  • Do not rely on the enrollment key if you already have a provisioned config.
  • Do not leave a player in maintenance after work is finished.
  • Do not expect the handoff page to last forever. It is expiring by design.
